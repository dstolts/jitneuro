# Context and Documentation Updates

- **Hub.md:** Update status as tasks complete. TodoWrite items MUST be mirrored in `<project>/.HUB/Hub-XX.md` under "ACTIVE TODO" section as soon as they are added to TodoWrite. This is the durable copy -- session state and in-memory TodoWrite lists are volatile and will be lost on context reset. When adding/completing/removing a TodoWrite item, update the Hub file in the same action. If no Hub file exists in the project's .HUB folder, create one.
- **Decision.md:** Log all decisions with date, reasoning in .HUB/Decision.md.
- **Engrams:** Update engrams at sprint completion when features are added, removed, or changed. Exception: update immediately if a story changes architecture or adds a major integration. Use /learn to evaluate and propose updates.
- **MEMORY.md:** Update brain when cross-project facts change (new integrations, new repos, infrastructure changes). MEMORY.md stores project-specific facts and confirmed patterns ONLY -- never duplicate rules or preferences already defined in CLAUDE.md or rules/. Repo .claude/CLAUDE.md files may intentionally repeat global rules as a safety net; that duplication is correct and should not be removed.
