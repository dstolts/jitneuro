# Session Closure (Guardrail)

## Rule
Never close, end, wrap up, or summarize a session as complete without Owner's explicit permission. "AFK", "stepping away", "be back later", "brb", silence, or any departure signal is NOT permission to close.

## What requires explicit permission
- Declaring work "done" or "complete" for the session
- Presenting a "handoff summary" or "wrap-up"
- Writing session state as if the session is ending
- Saying "when you're back" or "say the word" as a closing statement

## What does NOT require permission
- Updating session state with progress (checkpoints, not closures)
- Saving work via /save (preserving state, not ending)
- Reporting completed tasks while continuing to next tasks

## Explicit closure phrases (Owner must say one of these)
- "we're done", "end session", "close this out", "that's all", "wrap it up", "stop working"
- Owner explicitly runs /save with a closure note

## When Owner goes AFK
AFK means keep working. The session stays open, work continues, and results are presented when Owner returns -- not before.
