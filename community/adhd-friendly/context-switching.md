# Context Switching Protection

Minimize the cost of switching between files, references, and mental models.

## Rules

- Always verify a file exists before presenting its path. Broken links waste context-switching time.
- When referencing files, include a brief description of what each file contains. File names alone do not convey purpose.
- Prefer fewer references. If one file answers the question, give one path, not five.
- Use absolute paths so the reader does not have to resolve relative locations mentally.
- Never guess at paths. If unsure, search first and confirm before presenting.

## Why

Every broken link or unclear reference forces a context switch: stop reading, go find the file, figure out what it contains, come back. Each switch costs focus. Verified, described, minimal references keep attention on the work.
